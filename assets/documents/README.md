## üéØ How to Create a Great CV/Resume

### Essential Sections:
1. **Contact Information**
   - Name, Email, Phone, LinkedIn, GitHub
   - Portfolio website link

2. **Professional Summary**
   - 2-3 sentences about your expertise
   - Key skills and career goals

3. **Education**
   - Degree, University, GPA, Dates
   - Relevant coursework

4. **Work Experience**
   - Job title, Company, Dates
   - Bullet points with achievements (use action verbs)
   - Quantify results when possible

5. **Projects**
   - 2-3 key projects
   - Technologies used
   - Brief description and impact

6. **Skills**
   - Technical skills
   - Tools and technologies
   - Programming languages

7. **Certifications & Awards**
   - Relevant certifications
   - Academic or professional awards

## ‚úÖ CV Best Practices

**DO:**
- ‚úÖ Keep it to 1-2 pages
- ‚úÖ Use clear section headings
- ‚úÖ Include quantifiable achievements
- ‚úÖ Tailor to the job you're applying for
- ‚úÖ Use a professional font (Arial, Calibri, Times New Roman)
- ‚úÖ Proofread multiple times
- ‚úÖ Save as PDF to preserve formatting
- ‚úÖ Use action verbs (Developed, Implemented, Designed, etc.)

**DON'T:**
- ‚ùå Use fancy graphics (not ATS-friendly)
- ‚ùå Include a photo (unless required in your country)
- ‚ùå List every job/project you've ever done
- ‚ùå Use first-person pronouns (I, me, my)
- ‚ùå Include irrelevant information
- ‚ùå Use colors that don't print well in black & white
- ‚ùå Forget to update dates and contact info

## üìä ATS (Applicant Tracking System) Tips

Many companies use ATS to screen resumes. Make yours ATS-friendly:

1. **Use standard section headings**:
   - "Work Experience" not "Where I've Worked"
   - "Education" not "Academic Journey"

2. **Include keywords** from job descriptions

3. **Avoid:**
   - Tables and columns (can confuse ATS)
   - Headers and footers
   - Images and graphics
   - Unusual fonts

4. **Use standard file format**: PDF (unless DOC is specifically requested)
